## 1. 产品概述

工地运渣车管理系统是面向建筑施工企业、渣土运输公司的数字化管理平台，实现对运渣车辆全生命周期的智能化管理。系统通过整合车辆信息、运输调度、驾驶员管理、运输统计、安全监控等核心功能，解决工地渣土运输过程中存在的管理混乱、效率低下、安全隐患突出等问题，满足行业监管要求，提升企业运营效率。

- **核心目标**：实现运渣车辆全流程数字化管理，提升运输效率，降低安全风险
- **目标用户**：工地管理人员、运输调度员、安全员、企业管理者、系统管理员
- **市场价值**：帮助企业合规运营，降低运营成本，提升行业竞争力

## 3. 核心流程

### 3.1 运输任务调度流程

调度员创建运输任务 → 选择运输路线和渣土类型 → 指派空闲车辆和驾驶员 → 任务下发 → 驾驶员接单开始运输 → 系统实时跟踪位置和状态 → 任务完成 → 自动统计运输量。

```mermaid
flowchart LR
    A["创建运输任务"] --> B["选择路线和类型"]
    B --> C["指派车辆和驾驶员"]
    C --> D["任务下发"]
    D --> E["驾驶员接单"]
    E --> F["实时跟踪运输"]
    F --> G["任务完成"]
    G --> H["统计运输量"]
```

### 3.2 安全预警处理流程

系统实时监控车辆数据 → 检测到超速/疲劳驾驶 → 触发预警通知 → 安全员收到提醒 → 联系驾驶员处理 → 记录处理结果 → 生成违规记录。

```mermaid
flowchart LR
    A["实时监控数据"] --> B["异常检测"]
    B --> C{"是否违规"}
    C -->|是| D["触发预警"]
    C -->|否| A
    D --> E["通知安全员"]
    E --> F["处理违规"]
    F --> G["记录处理结果"]
```

### 3.3 车辆证件到期提醒流程

系统每日扫描证件有效期 → 检测即将到期证件 → 自动提醒管理人员 → 更新证件信息 → 记录更新历史。

## 4. 用户界面设计

### 4.1 设计风格

采用**工业/科技风**设计，体现工程管理的专业感和现代感。

- **主色调**：深海蓝 (#165DFF) - 代表专业、可靠
- **辅助色**：警示橙 (#FF7D00) - 用于预警提醒；成功绿 (#00B42A) - 用于正常状态；危险红 (#F53F3F) - 用于严重告警
- **中性色**：深灰 (#1D2129)、中灰 (#4E5969)、浅灰 (#C9CDD4)、背景色 (#F2F3F5)
- **按钮风格**：直角微圆角 (4px)，实线边框，悬停有轻微阴影和颜色变化
- **字体**：采用 "PingFang SC"、"Microsoft YaHei" 作为主字体，数字使用等宽字体增强可读性
- **布局风格**：卡片式布局，左侧导航栏 + 顶部工具栏 + 主内容区，层次分明
- **图标风格**：线性图标，简洁明了，统一的 24px 网格尺寸

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|-------|
| 仪表盘 | 数据概览 | 4个统计卡片（运输量、在线车辆、活跃任务、预警数），带渐变背景和图标 |
| 仪表盘 | 趋势图表 | 运输量趋势折线图、车辆状态饼图、任务完成率柱状图 |
| 仪表盘 | 预警列表 | 横向时间轴展示最新预警，不同严重程度用颜色区分 |
| 车辆管理 | 车辆列表 | 数据表格，状态标签用不同颜色，支持行展开查看详情 |
| 车辆管理 | 车辆表单 | 分区块表单，证件上传区域带拖拽效果 |
| 任务调度 | 实时跟踪 | 全屏地图容器，车辆图标带状态指示，信息弹窗 |
| 运输统计 | 可视化图表 | 可切换图表类型，支持数据钻取，悬停显示详细数据 |
| 安全监控 | 实时监控 | 卡片网格布局，异常卡片红色边框高亮，实时数据刷新动画 |

### 4.3 响应式设计

- **桌面端 (≥1200px)**：完整侧边栏 + 多列布局，充分利用屏幕空间
- **平板端 (768px-1199px)**：收起侧边栏为图标模式，表格可横向滚动
- **移动端 (<768px)**：底部导航栏，卡片单列布局，表格转置展示，触控优化

### 4.4 动效设计

- 页面加载：元素从下往上渐入，带延迟错位效果
- 数据更新：数字变化有滚动动画
- 预警提示：红色边框呼吸闪烁效果
- 按钮交互：悬停时轻微上浮 + 阴影变化
- 侧边栏展开/收起：平滑过渡动画
- 图表加载：数据从0到目标值的增长动画
